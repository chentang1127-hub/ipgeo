# IPGeo 30 天市场验证实验 v2

> 修正版 | 2026-06-26 → 2026-07-25
> 核心纠偏：这不是增长计划，是市场验证实验

---

## 目标重校

```
❌ 旧目标：300 免费 + 10 付费（虚荣数字）
✅ 新目标：
   1. 100 个真实开发者注册（接了 API key 试过至少一次）
   2. 10 个真实项目在调用（连续 7 天有请求）
   3. 3 个付费转化信号（点了 Upgrade 或发了邮件问价）
```

**为什么改：** 免费用户 ≠ 真实用户。一个注册了但从不调用的邮箱没意义。

---

## 核心策略修正

```
❌ 旧：社区曝光 → 刷存在感 → 等流量
✅ 新：搜索意图内容 + 开发者场景接入 → 被需要的人找到

增长引擎 = 技术内容（被 Google 收录）→ 开发者在搜索解决方案时自然发现 IPGeo
            +
           场景接入（pip install / npm install / curl 示例）→ 即装即用，立刻尝试
```

---

## Stack Overflow 的正确用法

```
❌ 旧理解：每天回答 2 个问题 → 带来流量
✅ 正确理解：

Stack Overflow 不是增长渠道，是 SEO 长尾载体。

一个高质量回答 =
  → 被 Google 收录
  → 几年后仍在排第一页
  → 开发者在搜索「怎么用 JS 获取访问者国家」时看到
  → 回答里有完整技术方案 + 末尾自然提到「我也做了个工具叫 IPGeo」

正确姿势：
  1. 回答完整技术方案（不推产品）
  2. 在"扩展"部分轻提 IPGeo
  3. 让 SEO 做长期分发

错误姿势：
  ✗ 广告式链接
  ✗ 直接贴 API
  ✗ 重复回答刷量
```

## 重写：30 天执行计划

### Week 1：种 SEO 种子（不是基建）

> **目标：2 篇技术内容上线 + 1 个可复制运行的 Demo**

**关键纠正：必须同步开始发内容，不能只准备。**

| 天 | 动作 | 产出 |
|----|------|------|
| 1 | **内容 1** "How to Get Visitor Country in JavaScript (3 lines)" — 纯技术教程，末尾自然提 IPGeo | 可被 Google 收录的教程 |
| 2 | **内容 1** 发布：网站 blog + Dev.to + Hashnode | 3 平台同时上线 |
| 3 | **内容 2** "IP Geolocation API: Start Free in 30 Seconds" — 复制粘贴即运行的快速入门 | 第二个 SEO 种子 |
| 4 | **内容 2** 发布 + **Demo 页优化** — 首页 Live Demo 改成「复制这段代码直接跑」的一键体验 | 降低试用摩擦 |
| 5 | **GitHub 仓库完善** — 三个仓库补 README 里的 curl 示例、补 social preview、加 topic 标签 | 社交证明就绪 |
| 6 | **API 目录提交** — apilist.fun, publicapis.io, freepublicapis.com 等 5 个 | 外链种子 |
| 7 | **Week 1 检查** — Google Search Console 是否已验证？sitemap 是否提交？ | SEO 基础就绪 |

**Week 1 产出：**
- 2 篇技术内容（博客 + Dev.to + Hashnode）
- 1 个优化后的 Demo 页
- 3 个 GitHub 仓库可 Star
- 5 个 API 目录提交
- Google 开始抓取

---

### Week 2：被 Google 收录（不是冷启动）

> **目标：Google 开始有 impressions + 第一批自然搜索流量**

| 天 | 动作 | 产出 |
|----|------|------|
| 1 | **Stack Overflow — 找到 3 个「有真实搜索量」的问题**（不是随便找，是 Google 搜索排名前 5 的问题） | 问题清单 |
| 2 | **回答第 1 个** — 完整技术方案 + 末尾自然引用 IPGeo | 一个 SEO 长尾资产 |
| 3 | **回答第 2 个** | 同上 |
| 4 | **回答第 3 个** | 同上 |
| 5 | **Reddit — 只发 1 篇**，r/SideProject: "I built a free IP geolocation API — looking for feedback" | 真实反馈，不是推广 |
| 6 | **检查数据** — Google Search Console impressions？哪个页面有展示？ | 数据反馈 |
| 7 | **Week 2 小结** — 哪些内容有 exposure？调整方向 | 进入 Week 3 前校准 |

**Week 2 产出：**
- 3 个高质量 SO 回答（SEO 长尾）
- 1 篇 Reddit 反馈帖
- Google Search Console 有数据

---

### Week 3：找有效渠道（不是扩量）

> **目标：找出哪个渠道真的带来注册。砍掉无效的。**

**这是最关键的一周。大多数渠道 = 0 流量。**

**每个渠道跟踪 3 个指标：**
- Impressions（曝光）
- Clicks（点击到网站）
- Signups（注册）

| 天 | 动作 |
|----|------|
| 1 | 检查所有渠道数据 — Dev.to views? SO upvotes? Reddit comments? Google impressions? |
| 2 | 如果 Dev.to 有流量 → 写第 3 篇。如果零 → 不写了。 |
| 3 | 如果 Google 开始有搜索词 → 针对那个词再写一篇。如果没有 → 等。 |
| 4 | 如果 Reddit 帖子有反馈 → 在那个 subreddit 继续参与。如果没有 → 换一个子版。 |
| 5 | 如果所有渠道都零 → 回到搜索意图。开发者到底在搜什么？ |
| 6 | 数据复盘：**只保留 TOP 1 渠道，其他全停。** |
| 7 | Week 3 小结 |

**Week 3 铁律：**
```
80% 渠道 = 0 流量
20% 渠道 = 全部流量

找到那个 20%，只做它。
```

---

### Week 4：复制有效路径（不是冲刺）

> **目标：把有效的动作重复 3 倍。不换平台，不试新东西。**

**前提：Week 3 找到了一个有效渠道。**

| 天 | 动作 |
|----|------|
| 1-7 | **只做一件事：把 Week 3 最有效的那个动作，重复做 3 倍。** |

**举例：**
```
如果 Dev.to "How to get visitor country in JS" 带来了 80% 注册 →
  → 本周写 3 篇同类教程：
     "How to get visitor country in Python (Flask)"
     "How to get visitor country in React"
     "How to block VPN users with IP geolocation"

不是：哦 Dev.to 有效但我同时去发 CSDN、知乎、V2EX
```

## 成功判断标准（30 天后）

```
✅ 成功信号（有一个就是赢了）：
  - Google 上搜 "ip geolocation api" 能看到 IPGeo
  - 有人把 IPGeo 接入真实项目（连续 7 天调用）
  - 有人点了 Upgrade 按钮或发邮件问付费

⚠️ 需要警惕：
  - 注册很多但零调用 → 产品体验有问题
  - 所有渠道都零流量 → 搜索意图没找准

❌ 失败信号：
  - 每天刷 SO 回答但 Google 没收录
  - 注册数在涨但全是假邮箱
  - 30 天后没法回答「开发者到底为什么用 IPGeo」
```

> **不要做增长计划，做市场验证实验。把种子种到搜索引擎里，找到那个真的带来注册的渠道，然后把所有资源砸进去。**
