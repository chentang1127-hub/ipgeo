API Documentation

IPGeo is a REST API. All responses are JSON. Authenticate with an X-API-Key header.

Authentication IP Lookup Batch Lookup My IP Usage Field Filtering Errors Rate Limits

Authentication

Include your API key in every request as the X-API-Key header. Get a key at /signup.

curl -H "X-API-Key: ipgeo_YOUR_KEY" https://api.getipgeo.com/v1/ip/me

Keys are prefixed with ipgeo_. If your key is missing or invalid, you'll receive a 401 Unauthorized response.

Single IP Lookup

GET/v1/ip/{ip}

Look up geolocation data for a single IPv4 or IPv6 address.

curl -H "X-API-Key: ipgeo_YOUR_KEY" https://api.getipgeo.com/v1/ip/8.8.8.8

Response:

{
  "ip": "8.8.8.8",
  "country": { "code": "US", "name": "United States" },
  "city": "Mountain View",
  "subdivisions": ["California"],
  "latitude": 37.4056,
  "longitude": -122.0775,
  "timezone": "America/Los_Angeles",
  "continent": { "code": "NA", "name": "North America" },
  "network": { "isp": "Google LLC", "asn": 15169 },
  "risk_flags": { "tor_exit": false, "hosting": true }
}

Response Fields

FieldTypeDescription

countryobjectcode (ISO 3166-1 alpha-2) + name (full name)
citystring | nullCity name (GeoIP2: ~95% accuracy; GeoLite2: lower)
subdivisionslist | nullAdministrative divisions (state / province)
postal_codestring | nullPostal / ZIP code
latitudefloat | nullApproximate latitude
longitudefloat | nullApproximate longitude
accuracy_radius_kmint | nullAccuracy radius in kilometers
timezonestring | nullIANA timezone identifier
continentobject | nullcode + name
networkobject | nullisp (provider name) + asn (AS number)
risk_flagsobjecttor_exit (bool) + hosting (bool) — present only when flagged
Pro & Business plans use MaxMind GeoIP2 databases for higher city-level accuracy (~95% vs ~37% for GeoLite2). Free & Starter plans use GeoLite2.

Batch Lookup

POST/v1/ip/batch

Look up up to 100 IPs in a single request. Each IP in the batch counts toward your quota.

curl -X POST https://api.getipgeo.com/v1/ip/batch \
  -H "X-API-Key: ipgeo_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"ips": ["8.8.8.8", "1.1.1.1", "2606:4700:4700::1111"]}'

Response:

{
  "results": [
    { "ip": "8.8.8.8", "country": "US", ... },
    { "ip": "1.1.1.1", "country": "AU", ... },
    { "ip": "2606:4700:4700::1111", "country": "US", ... }
  ]
}

Maximum 100 IPs per request. Returns 400 if exceeded.

My IP

GET/v1/ip/me

Look up the calling client's own IP address. No IP parameter needed.

curl -H "X-API-Key: ipgeo_YOUR_KEY" https://api.getipgeo.com/v1/ip/me

Usage

GET/v1/usage

Check your current billing period's usage and quota.

curl -H "X-API-Key: ipgeo_YOUR_KEY" https://api.getipgeo.com/v1/usage

Response:

{
  "plan": "pro",
  "used": 125430,
  "quota": 500000,
  "remaining": 374570,
  "period": "2026-06"
}

Field Filtering

Add ?fields= to limit which fields are returned. Useful for reducing bandwidth.

# Return only country and city
curl -H "X-API-Key: ipgeo_YOUR_KEY" \
  "https://api.getipgeo.com/v1/ip/8.8.8.8?fields=country,city"
{
  "ip": "8.8.8.8",
  "country": { "code": "US", "name": "United States" },
  "city": "Mountain View"
}
